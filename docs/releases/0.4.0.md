# Metastable Nucleation Suite 0.4.0

MNS 0.4.0 is a contributor, governance and publication-foundation release. It makes the project easier to review, contribute to, archive and cite; it does **not** promote the project to hardware-validated scientific status.

## Highlights

### Contributor onboarding and governance

- New bounded contributor onboarding path with contribution lanes by stack and role.
- Code of Conduct and private vulnerability-reporting policy.
- Architecture and contribution map for Python, semantic assets, .NET operations, Rust/WASM visualization and hardware boundaries.
- Structured onboarding issue form and corrected experiment/PBI issue forms.
- Pull-request guidance that permits `Not applicable` for scientific fields in documentation, CI and tooling changes.

### External contribution inventory

- Shared contribution-matching thread in issue #93.
- Initial set of bounded `good first issue` tasks covering Windows setup, coverage, deterministic demos, documentation checks, module indexing, environment diagnostics, terminology and clean-install validation.
- Focused independent-review bundles coordinated through issue #74.

### Persistent archiving and publication workflow

- `CITATION.cff` is expanded as the canonical software metadata source.
- Zenodo archiving is defined as the immutable DOI route for exact releases and the concept DOI route for the evolving project.
- arXiv is defined as the public preprint route for the software or methods manuscript.
- The workflow distinguishes Zenodo version DOI, Zenodo concept DOI, arXiv identifier, journal DOI, Git tag and commit SHA.
- DOI and arXiv placeholders are prohibited; identifiers are added only after real records exist.
- Publication work is coordinated through issues #103, #104 and #105.

### Outreach and research-software publication

- Six-month contributor-acquisition, review and publication plan.
- Positioning focused on executable protocols, explicit data contracts, semantic provenance and critical review.
- Candidate software-paper routes include JOSS, JORS and SoftwareX, subject to current venue requirements at submission time.

## Release and Zenodo sequence

Merging the aligned 0.4.0 metadata into `main` triggers `publish-version.yml`, which creates immutable tag `v0.4.0` and dispatches the release workflow. Because the repository is enabled in the maintainer's Zenodo GitHub integration, Zenodo should ingest the resulting GitHub release and issue a version DOI plus a project concept DOI.

The DOI badge and citation links must be added in a follow-up change using the real identifiers returned by Zenodo.

## Scientific boundaries

This release does not claim:

- completion of Stage 1 physical validation;
- observation of spontaneous Bell nonlocality or no-signalling violation;
- demonstration of a Nucleation-Encoded Chalcogenide Ensemble (NECE);
- validation of draft scalar or configurational capacity models;
- superiority over conventional memory, photonic or computing platforms;
- peer review merely because a DOI or arXiv identifier exists.

Continuous integration verifies consistency with implemented software contracts. It is not experimental evidence.

## Upgrade notes

There are no intended breaking changes to the standalone Python workflow or published data contracts in this release. The primary changes affect contributor experience, governance, citation metadata and publication planning.
