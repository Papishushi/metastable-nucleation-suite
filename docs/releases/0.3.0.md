# Metastable Nucleation Suite 0.3.0

MNS 0.3.0 is an operational and reproducibility release. It expands how experiments are executed, packaged, recovered and reviewed; it does **not** promote the project to hardware-validated scientific status.

## Highlights

### Distributed control plane

- Optional Kestrel control plane for durable experiment submission and recovery.
- Cross-process orchestration through Extend0.
- MetaDB-backed run and artifact indexes.
- Versioned control-plane run and artifact contracts.
- Docker Compose `distributed` profile and control-plane smoke validation.

### Portable release artifacts

- Self-contained .NET CLI archives for Windows, Linux and macOS on x64 and ARM64.
- Python wheel and source distribution validation.
- Multi-architecture OCI images for the platform CLI, scientific worker and control plane.
- SPDX SBOM generation, provenance attestations and SHA-256 checksums.
- Extend0 dependency and SBOM inventory checks in the release pipeline.

### Scientific review and communication

- `REVIEWING.md` provides narrow review entry points by scientific domain.
- Issue #74 records the open call for independent scientific and technical reviewers.
- The README now separates implemented software, planned hardware validation and open scientific drafts.
- The two coequal MNS programmes and the material-specific NECE branch are identified without presenting draft architecture as released functionality.

### Publication workflow

- Canonical version metadata remains synchronized across `VERSION`, `CITATION.cff` and .NET metadata.
- A version bump merged into `main` creates the matching immutable tag and dispatches the existing release workflow.
- GitHub generates the public release notes from the tagged history; this file preserves the scientific scope and boundaries of 0.3.0 inside the source tree.

## Scientific boundaries

This release does not claim:

- completion of Stage 1 physical validation;
- observation of spontaneous Bell nonlocality or no-signalling violation;
- demonstration of a Nucleation-Encoded Chalcogenide Ensemble (NECE);
- validation of the draft scalar or configurational capacity models;
- superiority over conventional memory, photonic or computing platforms.

The general Metastate Atlas architecture and the scalar/NECE models remain in draft PRs #59–#62 and are intentionally excluded from the stable 0.3.0 release until their scope, uncertainty and review requirements are resolved.

## Installation

Use `INSTALL.md` for release verification and platform-specific installation. Every published archive is accompanied by checksums; supported artifacts also include SBOM and provenance attestations.

## Upgrade notes

The standalone Python workflow remains available. The distributed control plane is optional and does not replace scientific ABoxes, datasets or event artifacts as the source of truth.
