# Code of Conduct

## Our standard

Metastable Nucleation Suite (MNS) is a technical and scientific collaboration. Participants must make it possible to challenge claims, assumptions, methods and code without attacking people.

Expected behaviour includes:

- giving technically grounded criticism and declaring uncertainty;
- distinguishing demonstrated errors from concerns that still need testing;
- welcoming negative results, replication failures and well-supported no-go recommendations;
- using inclusive, professional language;
- respecting privacy, attribution choices and conflicts of interest;
- helping newcomers find a bounded contribution rather than expecting repository-wide knowledge.

Unacceptable behaviour includes:

- harassment, threats, discrimination or personal attacks;
- publishing another person's private information without permission;
- misrepresenting credentials, affiliations, review independence or experimental evidence;
- pressuring reviewers to soften technically supported criticism;
- presenting simulated, inferred or illustrative outputs as measured evidence;
- deliberately disrupting issues, pull requests or review records.

## Scientific disagreement

Strong disagreement is allowed. Review the artifact, not the author. State the claim being challenged, why it matters, the assumptions used and a test or change that could resolve the disagreement.

Maintainers must not close substantive objections merely because continuous integration passes. Unresolved scientific objections should remain visible until they are answered, tested, scoped out or explicitly recorded as open.

## Enforcement

Conduct concerns should be reported privately to the maintainer at `danielmolinero111@cyberdude.com`. Do not include sensitive personal information in a public issue.

The maintainer may edit or remove comments, reject contributions, temporarily restrict participation or permanently ban participants whose behaviour is incompatible with this code. Enforcement decisions should be proportionate, documented privately where appropriate and should not be used to suppress good-faith technical criticism.

## Scope

This code applies in repository spaces and in public spaces where a person represents MNS or acts on behalf of the project.
